# Manus Agent Integration Guide: UE 5.6 Level Editor API

**Audience:** This document is for the Manus agent connected to the Perforce server. It describes the **only correct workflow** for using the **Level Editor API** to modify the SOH_VR project.

---

## 1. Terminology: Read This First

To prevent any misunderstanding, we will use these exact terms:

-   **The Level Editor API:** This is the Flask web application we built, running at `http://<host>:8080`. Its only job is to **inspect files** and **generate Python scripts** via its REST API. You interact with it by making `curl` requests.

-   **The Unreal Editor:** This is the official Epic Games Unreal Engine 5.6 application. This is where the `SOH_VR.uproject` is opened. Its only job is to **run the Python scripts** that the Level Editor API generates.

**You do not open the Level Editor API. You call it. You open the Unreal Editor. You run scripts inside it.**

---

## 2. The Unbreakable Rules

1.  **NEVER MODIFY FILES DIRECTLY.** You do not edit `.uasset` files. You do not edit `.umap` files. You do not edit `.py` files in the project. Your only write operation is to run a script inside the Unreal Editor.

2.  **NEVER PUSH PYTHON SCRIPTS.** The Python scripts generated by the Level Editor API are temporary. They are executed and then discarded. They are **never** checked into Perforce. The only files you submit to Perforce are the modified `.uasset` and `.umap` files that the **Unreal Editor** saves after a script is run.

3.  **NEVER ACT WITHOUT APPROVAL.** You must ask for and receive explicit user approval before (A) executing any script in the Unreal Editor and (B) submitting any files to Perforce.

---

## 3. The Correct Workflow

This is the only approved sequence of operations.

### **Phase 1: Inspection (using the Level Editor API)**

1.  **Pull Latest from Perforce:** Sync the `SOH_VR` workspace.
    ```bash
    p4 sync //depot/SOH_VR/...
    ```

2.  **Call the Level Editor API to Load an Asset:** Send the asset file from your local workspace to the Level Editor API.
    ```bash
    curl -X POST http://<level-editor-api-host>:8080/api/load \
      -H "Content-Type: application/json" \
      -d '{"path": "/path/to/your/p4/workspace/SOH_VR/Content/Maps/SL_Trailer_Logic.umap"}'
    ```

3.  **Call the Level Editor API to Inspect:** Use `GET` requests to the API to understand the asset. These are free actions.
    -   `GET /api/actors`
    -   `GET /api/functions`
    -   `GET /api/levels`

### **Phase 2: Script Generation (using the Level Editor API)**

4.  **Call the Level Editor API to Generate a Script:** This returns a Python script as a block of text. It does not run anything.
    ```bash
    curl -X POST http://<level-editor-api-host>:8080/api/script/generate \
      -H "Content-Type: application/json" \
      -d '{"domain": "actors", "method": "spawn", "params": {"class_name": "StaticMeshActor"}}'
    ```

5.  **Save the Returned Script Text:** Save the script from the JSON response to a temporary local file (e.g., `/tmp/approved_script.py`).

### **Phase 3: Execution (using the Unreal Editor)**

6.  **STOP. ASK FOR USER APPROVAL.** Show the user the contents of the temporary script file. Explain what it will do. Wait for a "yes".

7.  **Open the Unreal Editor:** Launch the editor with the `SOH_VR` project.

8.  **Execute the Script INSIDE the Unreal Editor:** Use the command line to run the approved script.
    ```bash
    # Path to the official Unreal Editor executable
    UNREAL_EDITOR="/path/to/UnrealEngine/Engine/Binaries/Linux/UE5Editor"

    # Path to the SOH_VR project file
    UPROJECT="/path/to/your/p4/workspace/SOH_VR/SOH_VR.uproject"

    # Path to the temporary, user-approved script
    SCRIPT_FILE="/tmp/approved_script.py"

    # This command opens the Unreal Editor and runs the script within its process.
    $UNREAL_EDITOR "$UPROJECT" -ExecutePythonScript="$SCRIPT_FILE"
    ```

### **Phase 4: Submission (using Perforce)**

9.  **The Unreal Editor has now modified the asset files.** The script has done its job.

10. **STOP. ASK FOR USER APPROVAL.** Report the results to the user. Show the changes in the Unreal Editor if possible. Confirm the script executed successfully by checking the logs. Ask for permission to submit the modified `.uasset` or `.umap` files to Perforce.

11. **Submit the Changed ASSET to Perforce:** After getting a "yes", check out and submit the `.uasset` or `.umap` file that the **Unreal Editor** just saved. **DO NOT SUBMIT THE .py SCRIPT.**
    ```bash
    p4 edit //depot/SOH_VR/Content/Maps/SL_Trailer_Logic.umap
    p4 submit -d "[JARVIS] Spawned StaticMeshActor as per user request."
    ```

## 4. The `JarvisEditor` C++ Plugin

-   **What it is:** A C++ extension for the **Unreal Editor**.
-   **What it does:** It allows Python scripts running *inside* the Unreal Editor to edit Blueprint graphs.
-   **When you need it:** Only when the task requires adding/removing Blueprint nodes or connecting pins.
-   **Installation:** This is a one-time setup that modifies the `SOH_VR` project. **You must ask the user for permission before installing it.** The steps are in the `README.md`.

This plugin is **part of the Unreal Editor environment**. It is not part of the Level Editor API. The Level Editor API will simply generate scripts that *happen to call* the plugin's functions if they are available in the running Unreal Editor.

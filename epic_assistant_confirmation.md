# Epic Developer Assistant Confirmation (UE 5.6)

**Date:** 2026-03-05
**Source:** https://dev.epicgames.com/community/assistant/unreal-engine/conversation

## Question
In UE 5.6, confirming the K2Node Extras blob serialization format for Blueprint graph nodes.

## Response (verbatim)

In Unreal Engine 5.6:

1. **UEdGraphNode::Serialize()** still serializes the pin array as an `int32 NumPins` followed by calling `SerializePin()` for each pin.

2. **UEdGraphPin::Serialize()** maintains the same serialization order:
   - OwningNode (FPackageIndex), PinId (FGuid), PinName (FName), PinFriendlyName (FText), SourceIndex (int32), PinToolTip (FString), Direction (uint8), PinType (FEdGraphPinType), DefaultValue (FString), AutogeneratedDefaultValue (FString), DefaultObject (FPackageIndex), DefaultTextValue (FText), LinkedTo array, SubPins array, ParentPin, RefPassThrough, PersistentGuid (FGuid), BitField (uint32).

3. For K2Node_CustomEvent and K2Node_Event, **UK2Node_EditablePinBase** still serializes a `TArray<FUserPinInfo>` immediately after the pin array.

4. There are **no changes** in this serialization format or order between UE 5.4, 5.5, and 5.6. The format remains stable and backward-compatible.

## Conclusion
Our parser and writer implementation is confirmed correct for UE 5.4 through 5.6.

## Follow-up: FEdGraphPinType Serialization

### Response (verbatim)

For UE 5.6, the FEdGraphPinType serialization follows this known field order and types:

1. `PinCategory` (FName)
2. `PinSubCategory` (FName)
3. `PinSubCategoryObject` (FPackageIndex)
4. `ContainerType` (uint8)

If `ContainerType == 4` (Map), then it serializes `PinValueType` struct, which contains:
- `Category` (FName)
- `SubCategory` (FName)
- `Object` (FPackageIndex)
- `bIsWeakPointer` (int32)

Then serializes the following bool flags as int32:
- `bIsReference`
- `bIsWeakPointer`

Next, a `FSimpleMemberReference` struct:
- `MemberParent` (FPackageIndex)
- `MemberName` (FName)
- `MemberGuid` (FGuid)

Finally the remaining bool flags as int32:
- `bIsConst`
- `bIsUObjectWrapper`
- `bSerializeAsSinglePrecisionFloat`

This matches the stable serialization pattern from UE 5.4 through 5.6 with no public changes reported.

### Conclusion
Our FEdGraphPinType implementation is confirmed correct for UE 5.4-5.6.

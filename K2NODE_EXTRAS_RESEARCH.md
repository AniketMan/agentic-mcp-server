## 1. Source Confirmed: Oculus UE 5.6 Fork

The serialization source was pulled from both `//UE5/5.4.4-release` and the Oculus 5.6 fork. A diff of `EdGraphPin.cpp` and `EdGraphNode.cpp` between the two versions shows **zero changes to any Serialize function**. The diffs are limited to non-serialization code (MakeLinkTo, BreakLinkTo, node comment stripping, diff comparison). The pin serialization format is identical across both versions.

The Extras blob is everything serialized by `UEdGraphNode::Serialize()` AFTER `Super::Serialize(Ar)`. The tagged properties (position, node GUID, etc.) are handled by UAssetAPI as the export's `Data[]` array. The Extras blob contains the pin array and any subclass-specific data.

All validation was performed against clean .umap files from the VR project running on the Oculus 5.6 branch (ObjectVersionUE5 = 1017 = OS_SUB_OBJECT_SHADOW_SERIALIZATION).

### 1.1 Full Inheritance Chain (Serialization Order)

```
UEdGraphNode::Serialize(Ar)
    Super::Serialize(Ar)                          // Tagged properties -> UAssetAPI Data[]
    // --- EXTRAS BLOB STARTS HERE ---
    UEdGraphPin::SerializeAsOwningNode(Ar, Pins)  // Pin array

UK2Node::Serialize(Ar)
    Super (UEdGraphNode)::Serialize(Ar)           // Pins (above)
    // No additional serialization

UK2Node_EditablePinBase::Serialize(Ar)
    Super (UK2Node)::Serialize(Ar)                // Pins (above)
    Ar << UserDefinedPins                         // TArray<FUserPinInfo>

UK2Node_Event::Serialize(Ar)
    Super (UK2Node_EditablePinBase)::Serialize(Ar) // Pins + UserDefinedPins
    // No additional serialization (just version fixups on load)

UK2Node_CustomEvent::Serialize(Ar)
    Super (UK2Node_Event)::Serialize(Ar)          // Pins + UserDefinedPins
    // No additional serialization (just version fixups on load)
```

### 1.2 SerializePinArray (OwningNode path)

```
int32 NumPins                                     // << ArrayNum
for each pin (i = 0..NumPins-1):
    SerializePin(Ar, pin, i, nullptr, OwningNode)
```

### 1.3 SerializePin (OwningNode path)

```
bool bNullPtr                                     // << bNullPtr (int32, 0 = valid pin)
if not null:
    UObject* OwningNode                           // << LocalOwningNode (FPackageIndex, int32)
    FGuid PinGuid                                 // << PinGuid (16 bytes: A, B, C, D as uint32)
    // then calls PinRef->Serialize(Ar)
```

### 1.4 UEdGraphPin::Serialize(Ar) -- THE CORE FORMAT

```
UObject* OwningNode                               // FPackageIndex (int32) - reference to owning node export
FGuid PinId                                       // 16 bytes (A, B, C, D as uint32)
FName PinName                                     // FName (int32 index + int32 number)
FText PinFriendlyName                             // FText (editor-only, complex serialization)
int32 SourceIndex                                 // (UE5 only, >= EdGraphPinSourceIndex)
FString PinToolTip                                // FString (int32 length + chars + null)
uint8 Direction                                   // 0 = EGPD_Input, 1 = EGPD_Output
FEdGraphPinType PinType                           // (see below)
FString DefaultValue                              // FString
FString AutogeneratedDefaultValue                 // FString
UObject* DefaultObject                            // FPackageIndex (int32)
FText DefaultTextValue                            // FText

// LinkedTo array (references to connected pins)
int32 NumLinkedTo
for each linked pin:
    bool bNullPtr                                 // int32
    if not null:
        UObject* LinkedOwningNode                 // FPackageIndex (int32)
        FGuid LinkedPinGuid                       // 16 bytes

// SubPins array
int32 NumSubPins
for each sub pin:
    (same as SerializePin format)

// ParentPin
bool bNullParentPin                               // int32
if not null:
    UObject* ParentOwningNode                     // FPackageIndex
    FGuid ParentPinGuid                           // 16 bytes

// ReferencePassThroughConnection
bool bNullRefPassThrough                          // int32
if not null:
    UObject* RefOwningNode                        // FPackageIndex
    FGuid RefPinGuid                              // 16 bytes

// Editor-only persistent data
FGuid PersistentGuid                              // 16 bytes
uint32 BitField                                   // flags packed into uint32
    bit 0: bHidden
    bit 1: bNotConnectable
    bit 2: bDefaultValueIsReadOnly
    bit 3: bDefaultValueIsIgnored
    bit 4: bAdvancedView
    bit 5: bOrphanedPin
```

### 1.5 FEdGraphPinType::Serialize(Ar)

```
FName PinCategory                                 // FName (int32 index + int32 number)
FName PinSubCategory                              // FName (int32 index + int32 number)
UObject* PinSubCategoryObject                     // FPackageIndex (int32)
uint8 ContainerType                               // 0=None, 1=Array, 2=Set, 4=Map
    if Map: FEdGraphTerminalType PinValueType     // (category + subcategory + object + bIsWeakPointer)
bool bIsReference                                 // int32 (serialized as bool)
bool bIsWeakPointer                               // int32 (serialized as bool)
FSimpleMemberReference PinSubCategoryMemberReference
    UObject* MemberParent                         // FPackageIndex (int32)
    FName MemberName                              // FName (int32 index + int32 number)
    FGuid MemberGuid                              // 16 bytes
bool bIsConst                                     // int32 (serialized as bool)
bool bIsUObjectWrapper                            // int32 (serialized as bool)
bool bSerializeAsSinglePrecisionFloat             // int32 (editor-only, serialized as bool)
```

## 3. Critical Serialization Details

### 3.1 FName Serialization in Extras

FNames in the Extras blob are serialized as `int32 NameIndex + int32 Number` where NameIndex is an index into the asset's name map. The `_0` suffix in UE display names maps to Number=1 (UE convention: display `_N` = stored Number=N+1, except `_0` which is Number=1, and no suffix = Number=0).

### 3.2 FPackageIndex for Object References

Object references (OwningNode, DefaultObject, PinSubCategoryObject, LinkedTo nodes) are serialized as `int32` FPackageIndex values:
- Positive values = export index + 1 (1-based)
- Negative values = import index (negated, 1-based: -1 = import[0])
- Zero = null

### 3.3 FText Serialization

FText has multiple serialization paths based on `historyType` (int8):

| historyType | Name | Format |
| :--- | :--- | :--- |
| -1 | HISTORYTYPE_NONE | flags(uint32) + type(int8) + hasCultureInvariant(int32) + optional FString |
| 0 | HISTORYTYPE_BASE | flags(uint32) + type(int8) + namespace(FString) + key(FString) + source(FString) |
| 11 | HISTORYTYPE_STRINGTABLEENTRY | flags(uint32) + type(int8) + tableId(FName) + tableKey(FString) |

Most Blueprint pins use HISTORYTYPE_NONE with hasCultureInvariant=0 (empty FText).

### 3.4 FString Serialization -- CRITICAL DISTINCTION

```
None (no string):     int32(0)                    // 4 bytes total
Empty string "":      int32(1) + 0x00             // 5 bytes total
Non-empty string:     int32(len+1) + chars + 0x00 // 4 + len + 1 bytes
Negative length:      UTF-16 encoding             // int32(-charcount) + charcount*2 bytes
```

**This was the last bug found.** UE writes empty strings as length=1 + null byte, NOT length=0. Length=0 means "no string at all" (null). The parser returns `None` for length=0 and `""` for length=1.

### 3.5 bool Serialization

`Ar << boolValue` serializes a bool as `int32` (4 bytes): 0 or 1.

### 3.6 WITH_EDITOR / WITH_EDITORONLY_DATA

The `!Ar.IsFilterEditorOnly()` check means PinFriendlyName, PersistentGuid, and BitField are only serialized in editor builds. Since .uasset/.umap files from the editor are editor builds, these fields WILL be present in our files.

## 5. UAssetAPI Patches Required

Two patches were made to UAssetAPI to achieve byte-perfect round-trips on Oculus 5.6 branch assets:

### 5.1 METADATA_SERIALIZATION_OFFSET Fix

UE 5.5+ introduced a MetaData section between GatherableText and Imports in the package header. UAssetAPI read the MetaDataOffset from the header but never read or wrote the section body. This caused a 53-byte loss on every Restaurant map.

**Fix:** Added `public byte[] MetaDataBytes` field to `UAsset.cs`. Read path captures MetaData bytes when the section is a separate block (not inside the name map). Write path writes them back at the correct position.

### 5.2 SavedHash / SectionSixOffset Visibility

These fields were declared `internal` in UAssetAPI, making them invisible to pythonnet. Changed to `public` so the Python bridge can access them.

## 6. Architecture Summary

Two completely separate workflows exist:

**Workflow A (Manus/Headless):** Direct binary editing via UAssetAPI. Reads .uasset/.umap files, modifies exports/imports/properties/K2Node Extras blobs, writes back. No Unreal Editor needed. This is what the Level Editor API exposes.

**Workflow B (Local AI + Editor):** JarvisEditor C++ plugin running inside the Unreal Editor. Uses `FBlueprintEditorUtils` and `FKismetEditorUtilities` to add nodes, connect pins, and compile Blueprints through the engine's own API. This is for local AI assistants with editor access.

Manus agents ONLY use Workflow A. They never touch the C++ plugin.
